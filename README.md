# Hotel Capital Stack Diagnostic — MVP V1

A confidential, hotel-specific debt-service triage tool by Modern Hospitality Solutions.

## Run it

Easiest: double-click **`Hotel-Capital-Stack-Diagnostic.html`** — a single self-contained file that runs fully offline (no install, server, or internet).

To edit/maintain, use the source version (keep the files together in one folder):

- `index.html` — page shell + styling; loads `engine.js` and `app.js`
- `engine.js` — the standalone, unit-tested calculation engine (single source of truth)
- `app.js` — application logic (flow, results, scenario workbench, exports, privacy)
- `data-retention-policy.md` — plain-English privacy policy

The standalone is generated by inlining `engine.js` and `app.js` into `index.html`.

## What's built

Three modes, switchable anytime:

- **Quick Check** — a 30-second, 4-input path (rooms, NOI, monthly payment, maturity) that returns the headline: cash after your loan payment (in dollars), DSCR, and a maturity flag. ~80% of the value for ~20% of the inputs.
- **Guide me (beginner)** — plain-English flow, "I don't know" allowed everywhere, assumptions step hidden (sensible defaults used).
- **Expert** — full model control including the refinance-assumptions step and lender/recourse/prepay/cap-rate fields.

Engine and analysis:

- Every sensitive field supports exact / range / estimate / "I don't know" / "prefer not to say"
- Deterministic engine: DSCR, **cash after debt service (plain dollars)**, debt yield, loan constant, required NOI, supportable debt (DSCR / debt-yield / LTV constrained), and **required NOI to refinance using the same binding logic** as supportable debt
- **Refinance gap as sources & uses** — folds in net capex/PIP, refinancing costs, and prepayment/defeasance, not just balance vs. proceeds
- The differentiator: required NOI lift translated into ADR, occupancy, RevPAR, revenue, and expense terms — now with **per-lever flow-through** (ADR ~90%, occupancy ~65%, expense 100%) and a **plausibility verdict** that uses your comp-set position
- **NOI guardrail**: asks whether NOI already nets a management fee + reserve, and flags likely overstatement
- New context captured: borrower liquidity (which fixes are realistic), market position, NOI trend, cap-rate→value path, floating-rate cap, prepayment penalty
- **One diagnostic that routes you** — a "Diagnostic scan" panel scores every dimension (coverage, leverage/refi, maturity, rate, capex, capital stack/equity cushion, personal exposure, distribution) red/amber/green/unknown so the owner never has to guess which "tool" they need
- **Capital stack & risk** (guided, all optional): a waterfall that shows who gets paid first and how much cushion your equity has at a given value; a plain-English "who's personally on the hook" exposure read (recourse, full/partial/bad-boy guarantees); loan-type good/risk/tactics (CMBS, bridge, SBA, bank, life co); and a default/workout sequence — all folded into the same triage, not a separate tool
- Primary-issue triage with a ranked factor list (operations, leverage, maturity, rate, capex, capital-stack, structural, exposure); per-section + overall confidence (treats "exact but blank" as not provided)
- Visible assumption log, scenario workbench (now shows cash-after-debt too), tailored capital-stack options, missing-info checklist, lender-conversation prep
- Watermarked owner memo (.html) + Print/Save-as-PDF; privacy: session-only, anonymous mode, opt-in local save, one-click delete; adaptive advisory CTA

## Deploy to Vercel (static, zero-config)

This is a static site — `index.html` at the repo root loads `engine.js` and `app.js`. No build step, no backend.

The repo is already initialized and committed locally. To publish under the GitHub account **moonman312** and deploy:

**Option A — GitHub CLI + Vercel (fastest):**

```bash
# from this folder
gh repo create moonman312/hotel-capital-stack-diagnostic --public --source=. --remote=origin --push

# then deploy (installs the CLI if needed)
npm i -g vercel
vercel --prod
```

**Option B — GitHub website + Vercel dashboard:**

1. Create a new empty repo at github.com/new under `moonman312` named `hotel-capital-stack-diagnostic` (don't add a README).
2. Push:
   ```bash
   git remote add origin https://github.com/moonman312/hotel-capital-stack-diagnostic.git
   git branch -M main
   git push -u origin main
   ```
3. Go to vercel.com → Add New → Project → Import the repo. Framework preset: **Other**. Leave build & output settings empty. Deploy.

Vercel will serve `index.html` automatically. `node_modules` is gitignored, so nothing extra is built.

## Deliberately NOT in V1 (per the doc — these are V2/V3)

Document upload & extraction, T-12 normalization, loan-document parsing, secure accounts,
saved scenarios, benchmarking, portfolio/multi-property views, and any hosted backend.
The privacy *behavior and written policy* are fully implemented now; server-side features
(encryption at rest, accounts) are described in the policy as future commitments.

## Key assumptions you may want to tune

In `app.js`, the `DEF` object holds the defaults used when a user skips a field:
target DSCR (1.25 / 1.35 / 1.45), refi rate (7.0 / 7.5 / 8.0%), amortization (25 yr),
min debt yield (9.5%), max LTV (65%), refinancing cost (1.5% of new loan), and
flow-through assumptions — blended 55%, ADR 90%, occupancy 65%. Set these to MHS house views.
